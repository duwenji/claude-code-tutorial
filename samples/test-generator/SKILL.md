# テスト生成スキル

## 概要

指定されたコードを分析し、網羅的なテストを自動生成します。ハッピーパスだけでなく、エッジケースや異常系も含めた実用的なテストを生成します。

## 対応テストフレームワーク

- Vitest（デフォルト）
- Jest
- Testing Library（React コンポーネント）

## 入力パラメータ

| パラメータ | 型 | 必須 | 説明 |
|-----------|------|------|------|
| `target` | string | ✅ | テスト対象のファイルパス |
| `framework` | string | ❌ | テストフレームワーク（デフォルト: "vitest"） |
| `coverage_target` | number | ❌ | 目標カバレッジ（デフォルト: 80） |
| `include_edge_cases` | boolean | ❌ | エッジケースを含めるか（デフォルト: true） |

## 手順

### Step 1: 対象コードの分析

`target` ファイルを読み込み、以下を抽出する：
- 公開されている関数・クラス・コンポーネント
- 各関数のシグネチャ（引数・戻り値の型）
- 依存関係（インポート）
- 非同期処理の有無
- エラーハンドリングのパターン

### Step 2: テストケースの設計

各関数について以下のケースを設計する：

**ハッピーパス**
- 正常な入力での期待される動作

**境界値**
- 空文字列・null・undefined
- 最小値・最大値
- 空配列・配列の1要素

**異常系**
- 無効な入力
- 型エラー
- ネットワークエラー・DB エラー（モック）

**非同期**
- 成功ケース
- 失敗ケース（reject / throw）
- タイムアウト

### Step 3: テストファイルの生成

対象ファイルと同じディレクトリに `<filename>.test.ts` を生成する。

### Step 4: テストの実行確認

生成後にテストを実行して、全テストが通ることを確認する：
```bash
npx vitest run <generated-test-file>
```

## 出力形式

```typescript
import { describe, it, expect, vi } from 'vitest';
import { 対象関数 } from './対象ファイル';

describe('対象関数', () => {
  describe('ハッピーパス', () => {
    it('正常な入力で正しい結果を返す', () => {
      // Arrange
      const input = ...;
      // Act
      const result = 対象関数(input);
      // Assert
      expect(result).toBe(...);
    });
  });

  describe('境界値', () => {
    it('null を渡すとエラーを投げる', () => {
      expect(() => 対象関数(null)).toThrow();
    });
  });

  describe('異常系', () => {
    it('無効な入力を受け取ると適切なエラーを返す', () => {
      ...
    });
  });
});
```

## テストケース

| # | 対象 | 期待結果 |
|---|------|---------|
| 1 | ユーティリティ関数（純粋関数） | 境界値を含む網羅的なユニットテスト |
| 2 | React コンポーネント | レンダリング・ユーザー操作・状態変化のテスト |
| 3 | API クライアント関数 | 成功・失敗・タイムアウトのモックテスト |
