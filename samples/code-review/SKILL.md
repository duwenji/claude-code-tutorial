# コードレビュースキル

## 概要

指定されたコードまたは現在のブランチ差分を4つの観点でレビューし、優先度付きの改善点を提示します。

## 対象

- PR マージ前のセルフチェック
- 新機能実装後の品質確認
- レガシーコードの評価

## 入力パラメータ

| パラメータ | 型 | 必須 | 説明 |
|-----------|------|------|------|
| `target` | string | ❌ | レビュー対象（ファイルパスまたは "diff"、デフォルト: "diff"） |
| `focus` | string[] | ❌ | フォーカスする観点（デフォルト: 全4観点） |
| `severity` | string | ❌ | 報告する重大度の下限（"critical"/"major"/"minor"、デフォルト: "minor"） |

## 手順

### Step 1: 対象の読み込み

- `target` が指定されていれば指定ファイルを読む
- 指定がなければ `git diff HEAD` で現在の差分を取得する

### Step 2: 4軸レビュー

以下の観点でコードを分析する：

**正確性**
- バグ・ロジックエラーはないか
- エッジケースのハンドリングは適切か
- 非同期処理のエラーハンドリングはあるか

**パフォーマンス**
- 不必要な再計算・再レンダリングはないか
- N+1 クエリはないか
- メモリリークの可能性はないか

**セキュリティ**
- 入力値の検証はあるか
- SQL インジェクション・XSS のリスクはないか
- 機密情報がハードコードされていないか

**保守性**
- 関数・ファイルが長すぎないか
- 命名は適切か
- 重複コードはないか

### Step 3: 優先度付き結果の出力

以下の形式で出力する：

## レビュー結果

### サマリー
- 重大度別件数：Critical N件 / Major N件 / Minor N件
- 総合スコア：XX/100

### 問題点

| ID | 重大度 | 観点 | 場所 | 問題 |
|----|--------|------|------|------|
| R1 | 🔴 Critical | セキュリティ | auth.ts:42 | SQL インジェクションの可能性 |
| R2 | 🟡 Major | パフォーマンス | UserList.tsx:18 | 毎レンダリングで API 呼び出し |
| R3 | 🟢 Minor | 保守性 | helpers.ts:55 | 関数が長すぎる（80行） |

### 修正案

各問題について具体的な修正案を提示する。

### 良い点

コードの良い部分も必ず1〜3点挙げる。

## テストケース

| # | 入力 | 期待結果 |
|---|------|---------|
| 1 | React コンポーネント（型安全性なし） | 型エラーの指摘 |
| 2 | 適切な TypeScript コード | 良い点のみ、または Minor のみ |
| 3 | セキュリティ問題を含む API エンドポイント | Critical として報告 |
